5.2

Решения · MX-1…MX-20

Зафиксированные решения с обоснованием. new — добавлено/уточнено в этой сессии.

MX-1
Один набор макроскриптов — много поверхностей
Меню, тулбар, quad, шорткаты — из макроскриптов «OmnibusCloud» (Customize UI), один источник. Не одно окно и не ribbon. Механизм — макроскрипты + cuiRegisterMenus, не C# ActionTable.
MX-2
WPF + Material
Max хостит WPF нативно; MaterialDesignInXAML. Не Avalonia (она у десктоп-клиента).
MX-3
Без ribbon
Ribbon в Max — единственный Modeling Ribbon; наша группа — докуемый тулбар.
MX-4
Фикс. модальные диалоги
ResizeMode=NoResize, только close. Размер фиксирован — меняется только рабочая область, окно не растёт.
MX-5
Render: close ≠ cancel
Закрытие окна не отменяет джобу — она идёт на ферме, прогресс уходит в статусбар.
MX-6
Статусбар
PushPrompt/ReplacePrompt (текст всегда) + опц. progressStart/Update/End (бар с Cancel).
MX-7
Settings — отдельный модал
Sidebar-навигация (General / Connection / Output / Diagnostics / About), Cancel / Save.
MX-8
Один Render вместо ~17 кнопок
validate / pack / upload / submit — внутренние шаги с прогрессом, не отдельные кнопки.
MX-9
Output — 2 оси
Image / Animation → 4 режима RenderStill / StillTiled / Frames / Video. «Export .blend» — не режим рендера, а команда Export.
MX-10
Target
Комбобокс групп; «Run on whole network» — чекбокс UseAllClients, виден только по праву CanRunOnAllClientsOption; включён → комбобокс выключен.
MX-11
Только OIDC new
OIDC / PKCE через системный браузер; API key из UI убран полностью.
MX-12
Один блокер за раз
Один actionable-блокер + фикс; вся диагностика свёрнута за «Details».
MX-13
Phase model
Фазы из статуса серверной джобы (+ транзитный Cancelling); портировано из Blender-бриджа.
MX-14
Тема следует за Max
Читаем через color manager (colorMan / IColorManager); тёмная = нейтральный серый Max, светлая = OmnibusCloud; дефолт тёмная.
MX-15
Один акцент
Muted lime #5BBF4A + navy текст #1A2B4C. Сырой #50FE30 как fill не использовать.
MX-16
Persistence
Per-OS-user через OutWit.Common.Settings, не синкается; мастер-тоггл «Remember last render settings».
MX-17
Плагин требует аккаунт new
Render и Export disabled до логина; всегда доступны только Sign in / Settings / Portal / About. Гейт живёт в меню/тулбаре.
MX-18
Export — две цели new
Blender .blend (дефолт — сервер конвертит DCC и возвращает как результат, самостоятельная ценность 3ds Max → Blender) + чистый DCC JSON (локально).
MX-19
Свои логи new
Rolling Serilog в %APPDATA%\OmnibusCloud\Logs; таб Diagnostics — Open logs folder + Open last log (зеркало клиентского DiagnosticsLauncher).
MX-20
Рендер — на стороне Blender new
Плагин экспортирует нейтральный DCC; сервер собирает .blend и рендерит Blender (Cycles). Рендерер 3ds Max и его препроцессинг не используются — никаких Arnold / denoise / passes в UI.
5.3

Принципы дизайна

Сквозные правила, по которым собирались все экраны.

Нативность хоста
Идиома Windows-диалога, не Blender-панели: фикс. размер, нижний бар кнопок, рабочая область свапается.
Честная встройка
Показываем UI в полном диалоге, а не вырванными группами. Каждое состояние — целое окно.
Никакого фейка
Не выдумываем параметры. Рендер Blender-side → нет 3ds-Max-рендер-настроек.
Минимальная копия
Убираем всё, что дублирует заголовок или отдельный экран; нет пояснений, когда контекст очевиден.
Зеркалим клиент
Логи, OIDC, VMs — повторяем проверенные паттерны OutWit.Cloud.Client, не изобретаем.
Один шаг за раз
Один понятный блокер + действие; детали и диагностика — за «Details».
Фон — в статусбар
Длинные джобы живут в статусбаре; закрытие окна не отменяет работу (close ≠ cancel).
Один акцент
Бренд — единственный лайм; severity-цвета отдельно. Тема — из color manager Max.
