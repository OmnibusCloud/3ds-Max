Статус-бар — прогресс при закрытом окне

После submit окно Render можно закрыть/свернуть — рендер удалённый, машина художника свободна. Статус едет в нативный нижний статус-бар Max (prompt line): строка OmnibusCloud · … + опциональный прогресс-бар + Cancel. Переоткрыть диалог из меню/тулбара — полная картина и результат.

3.1

Состояния в prompt line

Та же фазовая модель, что в Render-диалоге (Раздел 4.x), но в одну строку. Фазы — из серверного статуса job; строка обновляется на каждый poll-тик. Completed/Failed остаются в строке до клика/нового рендера.

Cancelling — транзитная строка «OmnibusCloud · Cancelling…» (Cancel дизейблится). Клик по действию: Cancel — отмена джоба; Open folder — открыть папку результата; Details — переоткрыть Render-диалог на состоянии Failed. Можно вообще не открывать диалог: одной строки хватает на весь happy-path.

3.2

Механизм

Статус-бар Max доступен из SDK. Текст — всегда, прогресс-бар — опционально.

AТекст в prompt line

Interface::PushPrompt() / ReplacePrompt() на каждый poll-тик. Работает асинхронно всегда — это основной канал статуса.

BПрогресс-бар (опц.)

progressStart / progressUpdate / progressEnd — встроенный бар Max с собственным Cancel. Заточен под синхронные циклы, поэтому к нашему async-мониторингу подключаем аккуратно либо опускаем.

CCancel

Из встроенного бара (B) — родной Cancel; либо кликабельное «Cancel» в строке (A), дёргающее отмену job на сервере.

Решение: строка (A) — обязательна и несёт статус + кликабельные действия; прогресс-бар (B) — опционально к v1 (можно ASCII-прогресс прямо в тексте: Rendering ▓▓▓▓░░ 59%). Источник истины фаз — серверный статус job, как и в диалоге. Переоткрытие диалога даёт полный Diagnostics/результат.
