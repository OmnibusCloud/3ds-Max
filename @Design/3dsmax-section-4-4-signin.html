Sign in — OIDC через системный браузер

Единственный путь входа: OIDC / PKCE через системный браузер (не встроенный webview, без API key). Запускается из меню «Sign in…». Пока браузер ведёт аутентификацию в OmnibusCloud Identity, Max держит небольшой модальный диалог ожидания; по приходе callback на loopback сессия активируется, и Render / Export разблокируются.

4.4.1

Поток входа

Четыре шага: меню → системный браузер (OmnibusCloud Identity) → Max ждёт callback → залогинен. Токен прилетает на локальный loopback-редирект.

1
Меню → Sign in…
Из signed-out меню. Render / Export пока серые.
2
Системный браузер
OIDC / PKCE на OmnibusCloud Identity (auth.omnibuscloud.com). Вход, при необходимости — регистрация.
3
Max ждёт callback
Loopback-редирект → обмен кода на токен. Диалог ожидания.
4
Залогинен
Аккаунт в меню, Render / Export активны (Разделы 1–2).

Механика: плагин поднимает локальный loopback-listener, открывает системный браузер на authorize-эндпоинт OmnibusCloud Identity (PKCE, ClientId OmnibusCloud), ловит редирект с кодом и меняет его на токен. Завязка на клиентскую модель — CloudVm.SignInCommand / RestoreSessionAsync; webview и API key не используются.

Успех → окно закрывается, меню/тулбар разблокированы. Таймаут / отмена → понятный повтор.
