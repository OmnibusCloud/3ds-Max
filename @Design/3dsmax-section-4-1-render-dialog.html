Render-диалог — Windows-диалог

Переделан как нативный модальный диалог: фиксированный размер, нижний бар кнопок, группы (Output, Target). Между фазами окно не меняет размер — свапается только рабочая область. Логина здесь нет: вход делается из меню (`Sign in…`), а пока не залогинен, пункты Render/Export в меню disabled — диалог не открыть. Открыт ⇒ сессия уже есть.

4.1.4

Target · по правам пользователя

4.1.5

Details / Diagnostics

Открывается из «Details…» в Render-диалоге — отдельный модальный диалог (фикс. размер). Здесь живёт всё, что не нужно в основном потоке: проверки сцены, глубокие параметры рендера, сводка, версии. Длинный контент скроллится; окно не меняет размер.

Что внутри
Validation — грид Severity / Message / Suggested action: ✓ OK, ⚠ Warning, ✕ Error (готовность к экспорту)
Scene — read-only сводка: разрешение, диапазон, камера, output; рендер — Blender (Cycles) на ферме
About — версии плагина/Max/SDK + last error (для саппорта)
Validate / Preflight — перепрогнать проверки вручную; Close — вернуться в Render

Почему отдельный диалог: чтобы Render-диалог не рос в высоту (правка из твоего фидбэка). Без рендер-настроек 3ds Max: рендер идёт в Blender (Cycles) на ферме — рендерер 3ds Max и его препроцессинг не используются, поэтому здесь только валидация экспорта, сводка сцены и версии. Warning «2 nodes unsupported for DCC export» → «Review / exclude» подсвечивает узлы, которые не уедут в нейтральный DCC.
